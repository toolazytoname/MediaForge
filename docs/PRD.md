# PRD — MediaForge 产品需求文档

> 版本 v1.0 | 2026-07-04 | 作者：架构师会话（Claude Fable 5）

## 1. 背景与目标

个人开发者希望借助 AI 构建一条**全自动内容生产流水线**，在 AI+自媒体时代成长为超级个体：

- 选题、创作、发布全流程自动定时执行，把人的时间解放出来
- 做矩阵账号（多平台、多账号）实现内容变现
- 内容形式覆盖**图文 + 视频**，平台覆盖**国内 + 国外**
- **硬性品质要求**：不产垃圾内容。产出必须有价值 / 有趣 / 有信息增量，创作者本人愿意署名转发
- 变现方式必须合规、可公开（"站着把钱挣了"）

## 2. 非目标（明确不做）

- ❌ 不做成通用 SaaS 工具卖给别人（至少 V1 不做）
- ❌ 不做搬运洗稿、买卖账号、刷量
- ❌ 不自研平台发布器（复用开源）
- ❌ 不做真人出镜依赖的内容形式
- ✅ 提供**本地 Web 控制台**（图形化管控整条流水线：看板、审核、排期日历、手动触发、成本监控），但不做多用户/权限系统（单人使用）

## 3. 内容策略

### 3.1 内容支柱（V1 两条）

| 支柱 | 说明 | 数据源 | 差异化 |
|------|------|--------|--------|
| P1: AI/科技日报解读 | 每日精选 3-5 条 AI 领域动态，给出解读和观点 | RSS、HN、GitHub Trending、arXiv | AI 深度加工 + 真实观点，非罗列 |
| P2: 开源项目测评 | 每周 2-3 个值得关注的开源项目深度测评 | GitHub Trending/Release | Claude 真的读代码，给出工程师级判断 |

> 支柱可扩展，但每次只增不改，写入 `config.yaml` 的 `pillars` 列表。

### 3.2 一料多吃（内容形式矩阵）

一份核心内容（canonical content）→ 派生多平台原生格式：

| 派生格式 | 平台 | 生产方式 |
|----------|------|----------|
| 长文（1500-3000字） | **微信公众号、今日头条**（图文主阵地） | LLM API 生成 + 模板渲染配图；头条为纯文本+图片格式 |
| 图卡（3-6张竖图） | 小红书 | LLM 产出结构化文案 + HTML 模板渲染截图 |
| Thread（5-10条） | X/Twitter | LLM 改写 + X 官方 API |
| 口播短视频（60-90s） | 抖音 / TikTok / YouTube Shorts / 视频号 | MoneyPrinterTurbo API |
| 数字人口播视频（V2） | 抖音 / 视频号 / TikTok | AIGCPanel（数字人+声音克隆）走 VideoEngine 抽象接入 |
| 英文版长文 | Medium / dev.to（后期） | LLM 翻译管道 |

> 注：同一篇长文在公众号和头条双发（头条允许非独家），一次创作两处收益；头条有流量分成，是图文最快的变现入口。
> 整条流水线**不依赖 Claude Code**，配好 API key 与平台凭据后独立定时运行。

### 3.3 品质标准（质量门禁依据）

三维评分，每维 1-10，由 LLM 以"挑剔编辑"角色评分：

1. **信息增量**：读者读完是否知道了原文之外的东西（背景、影响、对比、判断）
2. **趣味/可读性**：是否有钩子、节奏、人话，而非 AI 腔
3. **原创视角**：是否有明确的观点或立场，能否一句话说出"作者认为什么"

**门禁规则**：总分 < 24/30 自动丢弃；任意单维 < 6 自动丢弃；通过者进入人审。

### 3.4 AI 内容标识（合规红线）

所有产出默认携带 AI 生成标识（按各平台要求：显式声明文案/元数据标记）。这是不可关闭的默认行为。

## 4. 平台矩阵规划

### V1（M0-M4）：图文先行，3 平台跑通闭环

- 国内图文（主阵地）：**今日头条**（有流量分成，变现最快）+ **小红书**（图卡，反馈快）
- 海外：**X/Twitter**（thread，有现成 skill，官方 API 可用）
- 微信公众号在 M4 加入（有官方 API，但冷启动流量弱，先攒存货）

### V2（M5+）：扩展

- 国内：微信公众号、抖音、B站、视频号（视频形式启动）
- 海外：YouTube Shorts、TikTok、Medium

### 账号矩阵原则

- 同平台多账号 = 不同支柱/不同人设，**不发相同内容**（查重风控）
- 每账号独立浏览器 profile + 独立 cookie 存储
- 错峰发布：同平台账号间隔 ≥ 2h

## 5. 变现路径（按阶段叠加）

| 阶段 | 变现方式 | 前提 |
|------|----------|------|
| 0-3月 | 无（积累期），目标是数据 | 日更稳定 |
| 3-6月 | 平台流量分成（头条/公众号/YouTube）| 粉丝过门槛 |
| 6月+ | 联盟佣金（工具推广）、Newsletter 订阅 | 信任积累 |
| 6月+ | **数字人口播带货/好物分享**（抖音星图/精选联盟、视频号小店）| 数字人管线（V2）+ 账号过带货门槛 |
| 12月+ | 付费社群/知识产品；流水线本身开源+咨询 | 品牌成形 |

> 数字人带货注意：平台对数字人内容有明确标识与准入要求（抖音要求虚拟人注册报备、实名绑定），设计上数字人只作为 `VideoEngine` 的一种实现（AIGCPanel 等），形象/声音素材必须有授权（用自己的形象克隆最稳），带货话术走同一质量门禁避免夸大宣传违规。

## 6. 成功指标

- **M2 结束**：流水线每天自动产出 ≥ 1 篇过门禁内容，人审通过率 ≥ 60%
- **M4 结束**：双平台自动发布稳定运行 14 天无人工干预（除每日 10 分钟审核）
- **3 个月**：单平台粉丝 ≥ 1000，或单篇爆款（平台前 10% 表现）≥ 3 篇
- **门禁有效性**：被丢弃内容占比 40-70%（太低=门禁失效，太高=创作管道有问题）

## 7. 运行成本预算

| 项 | 估算 | 说明 |
|----|------|------|
| LLM API | $30-80/月 | 创作+门禁评分，用 Haiku 做初筛、Sonnet 做创作可控制在下限 |
| 视频生成 TTS/素材 | $10-30/月 | MoneyPrinterTurbo 用免费 TTS（edge-tts）可近零成本 |
| 服务器 | $0-10/月 | 本地 Mac 跑 cron 起步；后期 VPS |
| 总计 | **< $120/月** | 超出预算需要在 config 中显式调整 `budget.monthly_usd` |
