## 1. 产品边界（关键认知，纠正了原计划的假设）

| 产品 | 定位 | 做什么 | 不做什么 |
|---|---|---|---|
| **蚁小二** yixiaoer.cn | **分发/账号/数据** 一体化 SaaS，25~40+ 平台 | 一键发布、账号授权管理、数据统计、私信评论聚合、CLI | **不做图文创作** |
| **易撰** yizhuan5.com | 创作辅助平台（独立产品） | 创作灵感、全网热点、竞品雷达、AI 改写、文章质量检测 | 不做分发 |

**结论**：蚁小二只是**分发搬运工**；MediaForge 最值钱的 **AI 全自动创作流水线（选题→创作→门禁→审核）是蚁小二没有的**。因此「照抄蚁小二」= 只抄它强的三块（发布/账号/数据），创作保留 MediaForge 自动流水线特色。用户已拍板：**分模块对标**。

**创作模块形态（2026-07-10，用户问「手动+自动要不要单独独立出来」→ 结论：不独立）**：
- **一个模块 · 两个入口 · 一个出口**——「AI 自动生成」与「人工手写」都产出**同一张 `contents` 表的 `status=draft`**，之后共用门禁→审核→发布后半程。落 M11-G。
- 理由：① `contents` 靠状态机串全程，拆独立子系统 = 复制状态机或造桥接，撞 HARD_PARTS「禁 mock 状态机」；② 蚁小二外包易撰是因它无流水线，MediaForge 拥有流水线（差异化命根子），手动只是同漏斗多开人工入口；③ 独立仓库 = 双鉴权/双部署/数据同步，YAGNI。
- 契约桥接：`topic_id NOT NULL UNIQUE` → 手动创作先造 `source='manual'` 轻量 topic 再挂 Content；`canonical_path NOT NULL` → 人写 markdown 落 `output/.../canonical.md`。零 schema 改动。
- **易撰编辑器实抓结论（反面参照）**：易撰编辑器是**话题门控**的（必须从热点/作品点「AI 改写」才进，无空白编辑器入口，直连 `/app/editor` 弹回话题库）——服务它「蹭热点洗稿」定位。MediaForge 手动编辑器**不抄这个门控**，要「想写就能写」的空白草稿入口；只抄易撰的**排版工具栏 + 编辑/预览分栏**观感。

## 2. 蚁小二真实 IA（一手实抓）

**一级导航仅 7 项（极简）**：`主页 · 发布 · 账号 · 数据 · CLI · 私信评论 · 更多`

| 页面 | 路由 | 真实结构 |
|---|---|---|
| 主页 | `/web` | 「开始发布」大卡 + 已连续发布 N 天 + 授权账号入口 + 常用入口(易撰/抓词GEO/青豆云/浏览器发布助手/CLI) + 团队额度(账号点数/云流量/AI算力/素材库/发布次数) |
| 发布 | `/web/publish` | tab【发布记录｜草稿箱】+ [批量管理][导出][**新增发布**] + 筛选(发布人/作品类型/发布状态/发布模式) + 任务列表 |
| 账号 | `/web/accounts` | [添加账号] + 40+ 平台分类(抖音/快手/视频号/小红书/B站/公众号/头条…TikTok/YT/FB/IG) + 网络设置(代理) |
| 数据 | `/web/overview` | tab【仪表盘｜账号数据｜作品数据｜排行榜｜负责人数据】+ 时间(昨日/近7/14/30日/自定义) + 粉丝/作品/播放/互动总览 + 平台×账号排行榜 |
| CLI | `/web/openclaw` | 本地 AI 工具介绍/演示/安装/授权令牌 |
| 私信评论 | `/web/social` | 聚合多平台私信评论（青豆云） |

**发布方式三分**（手册）：本机发布(桌面托盘) / 浏览器发布(扩展/助手) / 云端发布(云手机, 付费)。发布流程：**新增发布 → 类型(视频/图文/文章) → 发布方式 → 选账号 → 批量填表(最多80账号任务) → 定时/发布**。

**视频创作入口多**（用户观察，已印证）：主页「浏览器发布助手」直连 可灵AI/即梦AI/海螺/Vidu 等 AI 视频工具的产物一键发布；图文创作则外链易撰。

## 3. MediaForge 现状 vs 蚁小二（对标差距）

MediaForge 当前 SPA 侧栏 **10 项**：图文创作/选题池/内容库/审核台/发布日历/发布记录/账号/数据/运行台/设置。

| 维度 | 蚁小二 | MediaForge 现状 | 差距/动作 |
|---|---|---|---|
| IA 复杂度 | 7 项极简、以「发布」为中心 | 10 项、创作/选题/审核占大头 | **精简 + 重心从"创作"移到"发布/账号/数据"** |
| 发布页 | 任务管理台(记录/草稿/新增/批量/筛选) | 发布日历+发布记录分裂，无"新增发布"统一入口、无草稿箱、无批量 | 合并重构为蚁小二式发布中心 |
| 账号页 | 40+ 平台授权中心 + 代理 | cookie 健康表(3~4 平台) | 平台网格化 + 授权引导 |

## 4. 难点攻坚：发布通道（全系统赚钱瓶颈，用户要"提前解决"的难点）

**行业死结**：国内平台(抖音/小红书/公众号/头条…)无个人友好的官方发布 API，只能浏览器自动化。三条技术路线：

| 路线 | 代表 | 风控特征 | 登录态 | 无人值守 | 稳定性 |
|---|---|---|---|---|---|
| A. headless 后端自动化 | **MediaForge 现状** M4-3/M5-2 | 高(无头特征明显) | 自管 cookie，易失效 | ✅ 可 | **最脆弱** |
| B. 浏览器扩展 | **Wechatsync 5.9k⭐ / MultiPost 2.8k⭐** = 蚁小二"浏览器发布" | 极低(真人浏览器真会话) | 复用用户已登录会话，免管 | ❌ 需浏览器开着(半自动) | **最稳** |
| C. 云端托管浏览器 | 蚁小二"云端发布"(付费) | 低 | 云端会话 | ✅ | 稳但要钱 |
| D. 官方 API | Postiz(海外) / MediaForge X-API(M4-2) | 无(合规) | OAuth | ✅ | 最稳(仅限有API平台) |

**开源实况**（一手 GitHub）：
- **Wechatsync**：5.9k⭐/Fork 990，v2 活跃，**adapter 架构 + 已带 `.claude/skills`**，国内 29+ 平台，浏览器扩展，MIT 免费。
- **MultiPost-Extension**：2.8k⭐，1.4.3 活跃，国内 10+ 平台扩展，不支持多账号矩阵。
- **Postiz**：AI 排期，self-host，**只走官方 OAuth、不 scrape**→ 只能 X/Bluesky/Mastodon/Discord 等有 API 的**海外**平台，国内做不了。AGPL-3.0。
- **MediaPublishPlatform**：doubao 文档吹为"主选"，但 GitHub 搜索仅 2 个弱 repo——**doubao 研究文档此处有夸大，不足采信**（与 evaluation-notes 既往发现一致）。

**建议的发布通道三层策略**（符合 CLAUDE.md"发布端复用开源、本仓库只写编排层"）：
1. **海外有 API 平台** → 官方 API（已有 X M4-2）/ 可选 Postiz。
2. **国内无 API 平台** → **引入 Wechatsync/MultiPost 浏览器扩展**作为"浏览器发布"通道，替代自写 headless（B 路线，最稳）。MediaForge 只做"生成 bundle → 交给扩展 → 收发布结果"的编排。
3. **兜底** → 保留现有 headless Playwright 作无人值守降级（A 路线）。

> ✅ **已拍板（2026-07-10）**：引入浏览器扩展（Wechatsync/MultiPost）做国内发布通道，headless 降级兜底。落 M11-0（开源集成评估）→ M11-E（集成，涉真发=高危、人工确认、不进自治流）。

> ⚠️ **工作区现状提示（2026-07-10）**：未提交的 `AppShell.vue` 已把侧栏改成蚁小二 9 项，但按「外壳完全照抄」做——把「创作」指向 `/roadmap/creation` 占位、选题池藏成「素材」、内容库/审核台移出导航。**这与「分模块对标」冲突**（把 MediaForge 创作核心从 UI 藏了）。M11-A 第一步即拨正：发布/账号/数据抄蚁小二，创作/选题/内容/审核作为真实一等公民保留在导航。

## 6. 我的核心意见（浓缩）

1. **不照抄蚁小二整体**，分模块对标：抄它的发布/账号/数据，保留自己的自动创作流水线（差异化命根子）。
2. **UI 不是瓶颈，发布通道才是**。M10 已把 UI 迭代到能用；再花力气抠 UI 像不像，边际收益递减。真正卡赚钱的是"发布一发就掉线/被风控"——应优先把发布通道从"自写 headless"升级到"复用成熟浏览器扩展"。
3. **IA 要做减法**。MediaForge 10 项侧栏比蚁小二 7 项还复杂，违背了参考蚁小二"降低学习成本"的初衷。
4. **先图文闭环、后视频**顺序对；但图文更卷、视频是红利，图文的定位是"用最小成本验证自动化流水线跑得通"，别指望它是主要利润源。
