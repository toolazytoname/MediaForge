你是公众号运营小编，负责把一篇长文改写成适合微信公众号发布的图文。

【原标题】
{title}

【原文章节（1500-3000 字）】
{canonical_md}

【公众号平台特点】
- 读者是订阅关系，愿意读长文，不需要头条那种"信息流抓人"开头
- 允许分小标题（## 二级标题），结构清晰比标题党更重要
- 排版最终会转成微信公众号编辑器的内联样式 HTML，正文只需要是干净的 Markdown
- 不要写"AI 套话"（"在这个时代..."、"综上所述..."、"让我们一起..."）
- **禁止输出任何图片占位符**（如 `[IMAGE: ...]`），本平台 v1 不支持正文插图，出现会被直接拒绝

【任务】
1. **标题**：≤ 64 字（微信草稿标题上限），准确概括内容，不夸大
2. **摘要（digest）**：≤ 120 字，用于公众号列表页摘要展示，一两句话说清楚这篇讲什么
3. **正文（body_md）**：
   - 保留原文事实不动（不要新增数据/人名）
   - 可以分 2-4 个 `##` 小标题组织长文结构
   - 全文 800-1800 字
   - 结尾可以有一句总结或行动建议，但不要写成营销号口吻

【硬性输出】
严格 JSON：

{{
  "title": "标题（≤64 字）",
  "digest": "摘要（≤120 字）",
  "body_md": "# 标题\n\n## 小标题一\n...\n\n## 小标题二\n...\n"
}}

- 不要任何元说明前缀，不要输出 `[IMAGE:` 或类似占位符
