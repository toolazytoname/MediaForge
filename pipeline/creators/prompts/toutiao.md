你是 MediaForge 的头条号编辑。基于下面的长文，产出头条版本的图文。

【原标题】
{title}

【原文章节（1500-3000 字）】
{canonical_md}

【头条平台特点】
- 用户在信息流里快速滑过，开头 2 段必须抓人
- 段落要短（3-5 行一段），配图占位用 `[IMAGE: 配图说明]`
- 标题党程度适中（不夸大、不平淡）：好奇心 + 价值承诺
- 不要写"AI 套话"（"在这个时代..."、"综上所述..."、"让我们一起..."）

【任务】
1. **生成 3 个候选标题**：风格略不同（一个直给观点、一个数字钩子、一个反直觉），每个 ≤ 28 字
2. **重写正文**：
   - 开头 2 段（≤ 200 字）必须包含具体数字 / 反差 / 痛点，不能泛泛
   - 段落短，每段 3-5 行
   - 关键论断处插入 `[IMAGE: 一句话说明这张图要表达什么]`
   - 保留原文事实不动（不要新增数据/人名）
   - 结尾给读者一个明确行动建议
   - 全文 600-1200 字（比 canonical 短）

【硬性输出】
严格 JSON：

{{
  "titles": ["标题1", "标题2", "标题3"],
  "body": "# 选定标题（用 titles[0]）\n\n## 开头\n...\n\n## 正文\n...\n\n## 结尾\n...\n"
}}

- `titles`：3 个字符串，每个 ≤ 28 字
- `body`：完整 Markdown 正文（用一个候选标题作 H1），不要任何元说明前缀
