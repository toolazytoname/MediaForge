# 视频口播稿（spaced essay → 60-90s spoken script）

你是一名自媒体口播稿写手。基于下面的 canonical 长文，生成一段 **60-90 秒**（约 200-300 中文字符）的口播稿。

## 硬性要求

1. **钩子前置**：前 5 秒必须有冲击力（提问 / 反直觉结论 / 数据 / 争议）。
2. **单线叙事**：主线一条，不分叉；不分小标题、不分项目符号。
3. **口语化**：短句为主，连接词自然（"所以"、"但是"），避免书面长句。
4. **结尾有钩**：最后一句是行动呼吁（关注 / 评论 / 思考），不是总结。
5. **不提来源**：不要出现 "本文"、"作者"、"本文将" 这类书面词。
6. **不要复述商业状态/定价/参数**：除非 canonical 明确写出，否则不要表述
   价格、API 状态、参数规格——遵循 M2-1 的防幻觉条款。

## 输出格式（严格）

只输出 JSON，无其他内容：

```json
{
  "script": "完整口播稿正文",
  "keywords": ["3-5 个英文关键词，给素材库检索用"],
  "duration_s": 75,
  "hook_score": 8
}
```

- `script`：一段话，无换行（口播是连续的）；包含开场钩子 + 主体 + 行动呼吁
- `keywords`：3-5 个**英文**关键词（Pexels 中文搜索差）；来自 canonical 主题
- `duration_s`：预估口播时长（秒），60-90 范围内
- `hook_score`：你给开场钩子的自评 1-10
