【任务】
1. **只列出真正存在、且足够严重会影响发布的问题**（最多 3 个，按严重度排序）。如果这篇文章整体质量过硬、挑不出真正的硬伤，
   `problems` 可以只有 0～2 个——**不要为了凑够三个而硬找问题**；数量不是及格线，质量才是。
2. 对每个问题给出：
   - `category`：fact（事实不一致/编造数字） / title（标题党/误导） / structure（结构混乱/缺主线） / tone（AI 套话/营销腔/无信息钩子） / html（标签不兼容） / image（配图无关） / risk（伦理/版权/医疗法律边界）
   - `severity`：low / medium / high / blocker
   - `message`：一句话说清楚是什么问题
   - `evidence`：从文中摘一句最关键的证据（≤80 字）
   - `suggestion`：该如何修（≤40 字；不要替作者写正文）
   - `autoFixable`：true 表示"我知道怎么一句话改完"；false 表示需要大改或不可改

【硬性输出】
只输出严格 JSON，不要任何前言/解释/代码块围栏。结构：

{{
  "problems": [
    {{
      "category": "fact|title|structure|tone|html|image|risk",
      "severity": "low|medium|high|blocker",
      "message": "...",
      "evidence": "...",
      "suggestion": "...",
      "autoFixable": true
    }}
  ],
  "summary": "一句话总结这篇的状态"
}}

【判断准则】
- fact 类问题默认 severity=blocker（事实错了重写没用，应直接 discarded 或人审）
- **fact 类问题的事实来源优先级：【原文素材】> 你自己的训练知识**。正文里的具体数字/日期/人名/版本号/技术细节，
  只要能在【原文素材】里逐字或明确同义找到依据，就不算事实错误——哪怕这个说法跟你自己知道的、或训练数据
  截止时间之前的信息不一致，也不能判 fact 问题（你的训练数据有截止时间，原文素材更新更及时，原文素材才是
  权威源）。只有当正文的说法与【原文素材】明确矛盾，或者【原文素材】完全没有出现过这个具体细节（且素材非空），
  才判定为 fact 问题
- 【原文素材】为空（显示"（无）"）时，才允许完全凭你自己的知识判断合理性，但此时应更保守——不确定就不要
  升到 blocker，优先 medium/high，除非是明显的硬伤（自相矛盾、常识性错误）
- 正文对【原文素材】的**合理转述、总结、同义替换**不算事实错误——比如原文说"A 和 B 编译进同一个模块、协同运行"，
  正文转述成"A 与 B 协同工作"，语义没有走样，不判 fact 问题。只有正文**新增了原文没有的具体数字/专有名词/
  时间先后顺序/因果关系**，或者与原文明确矛盾，才判定为 fact 问题——挑用词精确度的刺不算
- risk 类问题默认 severity=high 以上
- tone / structure / title / html 类问题可重写（severity ≤ high 时可重写，blocker 仍只能 discarded）
- 正文中的 `[IMAGE: ...]` 占位符是设计好的insert点，图片在门禁通过后才生成——占位符本身不是缺陷，
  不要因为"没有真实图片"而扣分
- 不要给低优先级问题凑数——这篇写得好的地方直接略过，不进 problems
- 不要给"建议加强"式的空泛意见——只列真正阻碍发布的硬伤
